What is this model?

This is a custom Bronco EDGE Score model built for Boise State softball. It converts every player's statistics into a single number — how many wins they add above a Top-100 RPI-level athlete — then connects that directly to the revenue sharing budget.

A baseline player is set at Top-100 RPI average performance — the standard BSU competes to exceed every season. EDGE of 0.0 = that level exactly. Positive EDGE = above it. Negative EDGE = below it.

Top-100 RPI baselines are weighted averages of power conference and upper mid-major programs: ERA 3.00, wOBA .360, FLD% .968, SO% 15.0%, BB% 8.0%. These reflect the competition level BSU targets as a Top-100 program.

EDGE is a starting point, not a verdict. It does not capture leadership, injury trajectory, academics, or team chemistry — all of which matter in real decisions.

Stat glossary

wOBA

Weighted On-Base Average

The best single offensive metric. Weights each outcome by actual run value — HR worth more than 2B, 2B more than 1B, 1B more than BB. Scale mirrors OBP: .360 = Top-100 RPI average, .390+ = good, .430+ = elite.

Top-100 RPI avg ≈ .360

S/M%

Swing & Miss Rate

% of swings that completely miss the ball. Lower is better for hitters. Top-100 RPI hitter threshold is 16%. Hitters above 16% receive a small EDGE penalty. For pitchers, higher S/M means more swing-and-miss ability.

Top-100 RPI threshold = 16%

ERA

Earned Run Average (per 7 innings)

Most familiar pitching stat but influenced heavily by defense and luck. Use xFIP and SIERA for true skill. Top-100 RPI programs average ERA of 3.00, reflecting elite pitching staffs.

Top-100 RPI avg = 3.00

xFIP

Expected Fielding Independent Pitching

Luck-neutral ERA estimator. Removes defense, adjusts HR using fly ball rate. When xFIP is well below ERA, the pitcher has been unlucky and should improve. Primary ERA estimator in pEDGE.

Top-100 RPI avg ≈ 3.00

SIERA

Skill-Interactive ERA

The most complete ERA estimator — accounts for K%, BB%, and batted ball profile together. When SIERA and xFIP differ, SIERA is generally the better long-term predictor.

SO% / BB%

Strikeout & Walk Rate per Batter

Rate-based — fairer than raw totals because they adjust for batters faced. SO% above 15.0% is rewarded in pEDGE; BB% above 8% is penalized. These are the primary pitching skill metrics.

Top-100 RPI avg: SO% 15.0%, BB% 8.0%

FIP

Fielding Independent Pitching

ERA estimator using only strikeouts, walks, and HR — outcomes fully controlled by the pitcher. FIP higher than xFIP = pitcher gave up HR at above-average rate (often luck). FIP lower than ERA = defense hurt her.

WHIP

Walks + Hits per IP

Baserunners allowed per inning. 1.00 = elite, 1.40 = average, 1.75+ = struggling. Measures command and contact management. Included as a penalty component in pEDGE.

oEDGE / dEDGE / pEDGE

EDGE Components

oEDGE = offensive value. dEDGE = defensive value (estimated from box score — treat as approximate). pEDGE = pitching value. Two-way players earn both hitting and pitching EDGE. Total EDGE = all components combined.

EDGE/400 & pEDGE/100

Rate-Normalized EDGE

Projects to a common standard for fair comparison: 400 PA for hitters, 100 IP for pitchers. Use rate for evaluation and player comparisons. Use raw EDGE for budget allocation.

EDGE formulas & baselines

Offensive EDGE

oEDGE = (wOBA − 0.360)/0.360 × 5.0
+ HR × 0.12 + SB × 0.05
− max(0, S/M% − 16) × 0.015

Baseline .360 = Top-100 RPI average. S/M% penalty applies only above Top-100 threshold (16%).

Defensive EDGE

dEDGE = (fld% − .968)/.040 × 1.5
+ (PO + AST)/200 × pos_mult
(excluded if 0 chances)

Fielding baseline .968 = Top-100 RPI average. Zero-chance players excluded from fielding% component to prevent bugs.

Pitching EDGE

pEDGE = (3.00−ERA)/3.00 × IP/55 × 3.0
+ (SO%−15.0) × 0.07
− (BB%−8.0) × 0.07
+ (3.00−xFIP)/3.00 × 1.2
+ W × 0.15 − WHIP × 0.4

ERA and xFIP baselines set to 3.00 — Top-100 RPI weighted average for elite D1 programs.

Budget formula

EDGE/400 = (Total EDGE / PA) × 400
pEDGE/100 = (pEDGE / IP) × 100

effEDGE = EDGE × (PA/25) [SS only]
= EDGE [all others]

alloc = floor
+ (max(effEDGE,0)/Σeffpool)
× (budget − floor × n)

Every active player gets the floor. Redshirt = $0. SS players (under 25 PA) compete for the variable pool with EDGE scaled by PA÷25 — they can still earn above floor if their scaled EDGE is positive. Remaining pool splits by effective EDGE share.

Year-in-school framework

EDGE is always evaluated in the context of a player's year in school. A Grade D freshman is a completely different situation than a Grade D senior. The model applies this framework in every player's Details panel and in the Roster Decisions tab.

FR — Freshman: Developmental piece

EDGE is informational, not a verdict. The question is ceiling and trajectory, not current output. 3 years of development runway remain. Evaluate tools and trend, not just the number.

SO — Sophomore: Proving ground

Year 1 adjustment is behind her. Year 2 production starts revealing whether the tools translate to sustained D1 performance. Yellow flags warrant attention; red flags require a plan.

JR — Junior: Production window is now

Peak development window. EDGE reflects what this player is. Strong production = core piece, start planning her replacement pipeline now. Below replacement = honest evaluation needed.

SR/GR — Senior: Final season

EDGE is the complete picture. Roster management focus shifts to what must be replaced. Below-replacement seniors departing can actually be roster upgrade opportunities.

Grade A — Elite (EDGE ≥ 2.5)

Top-tier D1 contributor. Highest retention priority. Premium allocation. Generates significant portal interest from other programs.

Grade B — Above average (EDGE 1.0–2.5)

Consistent above-replacement contributor. Core roster piece worth continued investment.

Grade C — Average/developing (EDGE 0–1.0)

At or slightly above replacement. Monitor trends. Development could push to Grade B.

Grade D — Below replacement (EDGE < 0)

Below freely available replacement level. Evaluate alternatives. Small-sample Grade D (<25 PA) should be weighted lightly.
